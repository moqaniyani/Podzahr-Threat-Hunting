# 🛡️ بسته شکار تهدید و مانیتورینگ گروه تهدید پیشرفته Agrius

این مخزن حاوی رول‌های شناسایی و پیکربندی‌های نظارتی آماده برای استقرار (Ready-to-Deploy) است که به طور اختصاصی جهت شکار، ردیابی و خنثی‌سازی تاکتیک‌ها و حملات مخرب مربوط به گروه تهدید پیشرفته **Agrius** طراحی شده‌اند.

گروه **Agrius** به عنوان یک بازیگر سایبری با اهداف تخریبی شناخته می‌شود که معمولاً از باج‌افزارهای ساختگی (مانند Apostle) به عنوان پوششی برای پنهان کردن عملیات‌های اصلی خود و تخریب کامل زیرساخت‌ها استفاده می‌کند. یکی از خطرناک‌ترین اقدامات این گروه در فاز دسترسی و سرقت هویت، تلاش برای استخراج مستقیم دیتابیس اکتیو دایرکتوری (`ntds.dit`) با استفاده از ابزارهای بومی سیستم‌عامل است (**MITRE ATT&CK T1003.003 - OS Credential Dumping: NTDS**).

## 📦 ساختار مخزن و راهنمای استقرار (Deployment Guide)

این پوشه، ابزارهای نظارتی لازم برای حفاظت از حساس‌ترین بخش زیرساخت دامین شما (دامین کنترلرها) را در برابر رفتارهای تاکتیکی گروه Agrius فراهم می‌کند.

### 1. فایل `sysmon-config-agrius.xml`
* **کاربرد:** این یک قطعه کد پیکربندی نویزگیری‌شده برای ابزار Microsoft Sysmon است که جهت ثبت رفتارهای مخرب مرتبط با استخراج دیتابیس هویتی دامین طراحی شده است. این فایل به طور ویژه رویداد **Process Creation (Event ID 1)** را با تمرکز روی فرآیند دستورات `ntdsutil` پایش می‌کند. برای جلوگیری از هشدارهای کاذب (False Positives)، رفتارهای مرتبط با سیستم‌های پشتیبان‌گیری استاندارد کاملاً در این فایل استثنا شده‌اند.
* **کجا اعمال شود:** 
  1. مطمئن شوید که ابزار Microsoft Sysmon روی دامین کنترلرهای (DC) اصلی و فرعی شبکه مستقر است.
  2. این قطعه کد XML را با فایل پیکربندی فعلی سیزمون خود روی سرورها ادغام (Merge) کنید.
  3. پیکربندی آپدیت‌شده را از طریق خط فرمان (با دسترسی Admin) اعمال کنید:
     ```bash
     sysmon.exe -c sysmon-config-agrius.xml
     ```

### 2. فایل `wazuh-rules-agrius.xml`
* **کاربرد:** یک رول اختصاصی وازو در بالاترین سطح خطر امنیتی (**Level 14**) با شناسه `Rule ID: 100601`. این رول لاگ‌های فرستاده شده از سیزمونِ دامین کنترلرها را تحلیل کرده و به محض شناسایی فرمان‌های استخراج دیتابیس هویتی (مانند ساخت کپی از طریق متد IFM)، فوراً یک هشدار بحرانی صادر می‌کند. این رول به گونه‌ای نویزگیری شده که فعالیت ابزارهای بکاپ‌گیری مانند Veeam باعث فعال شدن کاذب آن نشود.
* **کجا اعمال شود:**
  1. به سرور مرکزی یا وازو منیجر (Wazuh Manager) خود متصل شوید.
  2. این رول را در فایل رول‌های محلی و سفارشی سیستم در مسیر زیر کپی کنید:
     `/var/ossec/etc/rules/local_rules.xml`
  3. صحت سینتکس کانفیگ را بررسی کرده و سرویس وازو منیجر را ریستارت کنید:
     ```bash
     /var/ossec/bin/wazuh-control restart
     ```

## ⚡ پیشنهاد عملیاتی: پاسخ خودکار (Active Response)

با توجه به حساسیت فوق‌العاده بالای سرقت دیتابیس دامین، شدیداً توصیه می‌شود که رول شناسه `100601` را به ماژول **Wazuh Active Response** متصل کنید تا به محض تلاش برای دامپ پسوردها، پروسس مربوطه در کسری از ثانیه متوقف شده و از خروج اطلاعات حیاتی شبکه دامین جلوگیری شود.
