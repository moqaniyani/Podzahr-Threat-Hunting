
# 🛡️ بسته شکار تهدید و مانیتورینگ گروه Ajax Security Team

این مخزن حاوی رول‌های شناسایی و پیکربندی‌های نظارتی آماده برای استقرار (Ready-to-Deploy) است که به طور اختصاصی جهت شکار، ردیابی و خنثی‌سازی تاکتیک‌ها و حملات جاسوسی سایبری مربوط به گروه **Ajax Security Team** طراحی شده‌اند.

گروه **Ajax Security Team** (معروف به اپریشن Saffron Rose) یکی از بازیگران باسابقه در حوزه جاسوسی سایبری و سرقت اطلاعات (Data Stealing) است. این گروه به طور معمول از مهندسی اجتماعی پیشرفته برای ترغیب کاربران به اجرای بدافزارهای کاستومایز شده خود استفاده می‌کند. یکی از تاکتیک‌های شاخص آن‌ها پس از نفوذ، تزریق کدهای مخرب به درون فرآیندهای مرورگرهای معتبر (مانند گوگل کروم و فایرفاکس) جهت سرقت مخفیانه کوکی‌ها، اطلاعات سشن و اطلاعات هویتی کاربران است (**MITRE ATT&CK T1055.002 - Process Injection: Source Engine Injection**).

## 📦 ساختار مخزن و راهنمای استقرار (Deployment Guide)

این پوشه، ابزارهای نظارتی لازم برای حفاظت از سیستم‌های کلاینت و سرورهای دسترسی کاربران را در برابر رفتارهای تاکتیکی سرقت اطلاعات این گروه فراهم می‌کند.

### 1. فایل `sysmon-config-ajax.xml`
* **کاربرد:** این یک قطعه کد پیکربندی نویزگیری‌شده برای ابزار Microsoft Sysmon است که جهت ردیابی رفتارهای مربوط به تزریق کد به مرورگرها طراحی شده است. این فایل رویداد **CreateRemoteThread (Event ID 8)** را پایش می‌کند تا هر زمان فرآیند ناشناسی بخواهد رشته‌ای را به مرورگرها تزریق کند، آن را لاگ کند. فرآیندهای قانونی سیستم‌عامل مانند `csrss.exe` و رفتارهای خود‌ارجاعی مرورگرها کاملاً برای جلوگیری از هشدارهای کاذب (False Positives) استثنا شده‌اند.
* **کجا اعمال شود:** 
  1. مطمئن شوید که ابزار Microsoft Sysmon روی سیستم‌های کاربران (Endpoints) و سرورهای پرخطر مستقر است.
  2. این قطعه کد XML را با فایل پیکربندی فعلی سیزمون خود ادغام (Merge) کنید.
  3. پیکربندی آپدیت‌شده را از طریق خط فرمان (با دسترسی Admin) اعمال کنید:
     ```bash
     sysmon.exe -c sysmon-config-ajax.xml
     ```

### 2. فایل `wazuh-rules-ajax.xml`
* **کاربرد:** یک رول اختصاصی وازو در سطح خطر امنیتی بالا (**Level 12**) با شناسه `Rule ID: 100701`. این رول به عنوان تحلیل‌گر متمرکز، لاگ‌های Event ID 8 سیزمون را بررسی می‌کند و به محض کشف تزریق رشته از یک منبع غیرمجاز به مرورگر، آلارم بحرانی صادر می‌کند. شرط منفی همنام بودن منبع و مقصد، تضمین می‌کند که کارهای عادی خود مرورگر باعث ایجاد آلارم کاذب نشود.
* **کجا اعمال شود:**
  1. به سرور مرکزی یا وازو منیجر (Wazuh Manager) خود متصل شوید.
  2. این رول را در فایل رول‌های محلی و سفارشی سیستم در مسیر زیر کپی کنید:
     `/var/ossec/etc/rules/local_rules.xml`
  3. صحت سینتکس کانفیگ را بررسی کرده و سرویس وازو منیجر را ریستارت کنید:
     ```bash
     /var/ossec/bin/wazuh-control restart
     ```

## ⚡ پیشنهاد عملیاتی: پاسخ خودکار (Active Response)

با توجه به اینکه حملات استیلر اطلاعات سشن را در ثانیه به سرقت می‌برند، پیشنهاد می‌شود رول شناسه `100701` را به یک ماژول **Active Response** متصل کنید تا در صورت تشخیص تزریق، فرآیند مخرب مبدأ (بدافزار استیلر) فوراً توسط اِیجنت وازو کشته (Kill) شود.
