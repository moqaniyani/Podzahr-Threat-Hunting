# 🛡️ بسته شکار تهدید و مانیتورینگ گروه تهدید پیشرفته ALLANITE

این مخزن حاوی رول‌های شناسایی و پیکربندی‌های نظارتی آماده برای استقرار (Ready-to-Deploy) است که به طور اختصاصی جهت شکار، ردیابی و خنثی‌سازی تاکتیک‌ها و حملات جاسوسی زیرساختی مربوط به گروه تهدید پیشرفته **ALLANITE** طراحی شده‌اند.

گروه **ALLANITE** به عنوان یک بازیگر سایبری متمرکز بر زیرساخت‌های حیاتی (به ویژه بخش انرژی و نیروگاه‌های برق) شناخته می‌شود. این گروه به طور معمول از تکنیک‌های شناسایی بومی برای استخراج اطلاعات شبکه و کشف فایل‌های حاوی گواهی‌ها و پسوردهای ادمین استفاده می‌کند. یکی از تاکتیک‌های شاخص آن‌ها، سوءاستفاده از ابزار بومی و پیش‌فرض سیستم‌عامل جهت جستجوی متنی عبارات حساس در فایل‌ها و لاگ‌ها است (**MITRE ATT&CK T1083 - File and Directory Discovery**).

## 📦 ساختار مخزن و راهنمای استقرار (Deployment Guide)

این پوشه، ابزارهای نظارتی لازم برای حفاظت از سیستم‌های لبه و سرورهای مدیریتی را در برابر رفتارهای تاکتیکی شناسایی و جمع‌آوری اطلاعات این گروه فراهم می‌کند.

### 1. فایل `sysmon-config-allanite.xml`
* **کاربرد:** این یک قطعه کد پیکربندی نویزگیری‌شده برای ابزار Microsoft Sysmon است که جهت ثبت رفتارهای مخرب مرتبط با جستجو و کشف فایل‌های حساس طراحی شده است. این فایل به طور ویژه رویداد **Process Creation (Event ID 1)** را با تمرکز روی فرآیند دستورات مشکوک ابزار `findstr` پایش می‌کند. برای جلوگیری از هشدارهای کاذب (False Positives)، رفتارهای مرتبط با اسکریپت‌های قانونی لاگین ویندوز و پراب‌های مانیتورینگ سازمان کاملاً استثنا شده‌اند.
* **کجا اعمال شود:** 
  1. مطمئن شوید که ابزار Microsoft Sysmon روی سرورها و کلاینت‌های حیاتی شبکه مستقر است.
  2. این قطعه کد XML را با فایل پیکربندی فعلی سیزمون خود ادغام (Merge) کنید.
  3. پیکربندی آپدیت‌شده را از طریق خط فرمان (با دسترسی Admin) اعمال کنید:
     ```bash
     sysmon.exe -c sysmon-config-allanite.xml
     ```

### 2. فایل `wazuh-rules-allanite.xml`
* **کاربرد:** یک رول اختصاصی وازو در سطح خطر بالا (**Level 11**) با شناسه `Rule ID: 100801`. این رول لاگ‌های فرستاده شده از سیزمون را تحلیل کرده و به محض شناسایی فرمان‌های جستجوی متنی پسوردها یا کانفیگ‌ها توسط کاربران غیرمجاز، فوراً آلارم صادر می‌کند. شرط منفی روی پردازش‌های والد (`parentImage`) تضمین می‌کند که کارهای دوره‌ای سیستم‌عامل باعث ایجاد هشدار کاذب نشود.
* **کجا اعمال شود:**
  1. به سرور مرکزی یا وازو منیجر (Wazuh Manager) خود متصل شوید.
  2. این رول را در فایل رول‌های محلی و سفارشی سیستم در مسیر زیر کپی کنید:
     `/var/ossec/etc/rules/local_rules.xml`
  3. صحت سینتکس کانفیگ را بررسی کرده و سرویس وازو منیجر را ریستارت کنید:
     ```bash
     /var/ossec/bin/wazuh-control restart
     ```

## ⚡ پیشنهاد عملیاتی: پاسخ خودکار (Active Response)

جهت مقابله اثربخش با فاز سارق اطلاعات این گروه، پیشنهاد می‌شود رول شناسه `100801` را به ماژول **Wazuh Active Response** متصل کنید تا به محض تلاش یک یوزر تعاملی یا وب‌شِل برای اسکن متنی فایل‌های حساس، سشن کاربر یا خط فرمان مربوطه فوراً مسدود شود.
